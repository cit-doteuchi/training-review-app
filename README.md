# 研修復習ログアプリ

## 概要

研修で復習した内容を記録・管理するためのコンソールアプリです。
復習項目を登録し、一覧表示、詳細表示、編集、削除、成果サマリーの確認ができます。

## 作成目的

研修で学んだことを記録し、あとから振り返れるようにするために作成しました。
また、研修期間中に作成した成果物として、計画的にアプリを制作した過程を示すことも目的としています。

## 主な機能

- 復習項目の登録、編集、削除
- 復習項目の一覧表示と詳細表示
- 成果サマリーの表示
- CSVへの保存と読み込み

## 使用技術

- Java
- 標準ライブラリ
- コレクション（ArrayList、List、Map、LinkedHashMap）
- ファイル入出力（BufferedWriter、Files、Path）
- CSV形式でのデータ保存

## 実行方法

プロジェクト直下で以下のコマンドを実行し、Javaファイルをコンパイルする。

　`javac -encoding UTF-8 src\*.java`

コンパイル後、以下のコマンドでアプリを実行する。

　`java -cp src Main`

アプリ起動後は、コンソールに表示されるメニュー番号を入力して操作する。

## データ保存について

起動時に data/review_items.csv から自動でデータを読み込みます。
「保存して終了」を選択すると、現在の復習項目をCSVファイルへ保存します。

## 工夫した点

### クラスごとの責務分担

アプリの処理を役割ごとにクラスを分けて実装しました。
Main はメニュー表示、ユーザー入力、処理の呼び出しを担当し、ReviewManager は復習項目の追加・検索・編集・削除・集計を担当する形にしました。
また、CSV保存・読み込み処理は ReviewCsvRepository に分離し、ファイル操作の処理を Main や ReviewManager に直接書かないようにしました。
これにより、ユーザーとのやり取り、データ管理、ファイル保存処理の役割が分かれ、後から機能を追加・修正しやすい構成にできました。

### ドキュメントによる進捗管理

実装前に requirements.md、task_list.md、progress_log.md、issue_log.md を作成し、要件、タスク、進捗、課題を分けて管理しました。
task_list.md では、作業を Phase ごとに分け、各タスクに優先度・状態・完了条件を設定しました。
progress_log.md では、作業した日ごとに「今日やること」「作業内容」「完了したこと」「次回やること」を記録しました。
これにより、次に何を実装するべきかを確認しながら、計画的に制作を進めることができました。
また、制作過程を記録として残すことで、完成したアプリだけでなく、どのように考えて作ったかも説明しやすくなりました。

### enumによるメニュー分岐

メニュー選択処理では、入力された文字列をそのまま switch 文で判定するのではなく、MenuOption enum に変換してから分岐する形にしました。
最初は "1" や "2" のような文字列で処理を分けていましたが、REGISTER、LIST、DETAIL、EDIT、SUMMARY、DELETE、EXITなどの名前を持つenumを用意し、入力値と処理内容の対応が分かりやすくなるようにしました。
これにより、メニュー項目が増えた場合でも、どの選択肢がどの処理に対応しているかを読み取りやすくなりました。

### Mapを使った成果サマリー

成果サマリーでは、総登録件数だけでなく、カテゴリ別件数と理解度別件数を表示できるようにしました。
カテゴリ別件数では、カテゴリ名をキー、件数を値として Map に保存し、登録済みの復習項目を1件ずつ確認しながら件数を集計しました。
理解度別件数では、理解度1〜5を先に0件で初期化し、各復習項目の理解度に応じて件数を加算する形にしました。
これにより、どの分野をどれだけ復習したか、どの理解度の項目が多いかを数字で確認できるようになり、学習成果を振り返りやすくなりました。

### CSV保存・読み込みによるデータ永続化

登録した復習項目をアプリ終了後も保持できるように、CSVファイルへの保存・読み込み機能を実装しました。
保存時は、ReviewItem の一覧を CSV の1行ずつに変換し、data/review_items.csv に書き込むようにしました。
読み込み時は、CSVファイルのヘッダー行を飛ばし、2行目以降を ReviewItem に変換して ReviewManager に追加する形にしました。
また、初回起動時などCSVファイルが存在しない場合は、空のリストを返してアプリを開始できるようにしました。
これにより、保存して終了し、次回起動時に前回のデータを復元できるようになりました。
