## 2026-05-11

### 今日やること

ReviewItemに持たせる項目を決める。

### 作業内容
ReviewItemに持たせる項目を検討し、復習内容を保管する置き場としてシンプルに運用できるよう、項目を決定した。
当初候補にあった「復習済みかのチェック」「できるようになったことの要約」「次に勉強すること」は、復習タスクの進捗管理や成果報告を細かく行うための項目であり、今回の「復習内容を保管する」という目的には必須ではないため採用しないことにした。

#### ReviewItemに持たせる項目

| 項目名 | 型 | 必須 | 内容 | 使用目的 |
|---|---|---|---|---|
| id | int | 必須 | 復習項目を識別する番号 | 詳細表示・更新対象の指定に使う |
| date | LocalDate | 必須 | 復習項目を登録した日 | 日付別に見返すために使う |
| category | String | 必須 | Java、SQL、HTML/CSSなどの分類 | カテゴリ別に整理するために使う。カテゴリ基準が不明なのでStringで管理 |
| title | String | 必須 | 復習項目のタイトル | 一覧表示で内容を把握するために使う |
| memo | String | 任意 | 復習内容のメモ | 後から内容を見返すために使う |
| understanding | int | 任意 | 理解度。1〜5で管理する | 必要に応じて理解度別に振り返るために使う |

### 完了条件

- [x] 各項目の型が決まっている
- [x] 必須項目と任意項目が分かれている

## 2026-05-12

### 今日やること

Main、ReviewItem、ReviewManager のクラス構成と役割を決める。

### 作業内容

アプリのクラス構成について検討した。  
当初は Main、ReviewItem、ReviewManager の3クラス構成を想定していたが、CSV保存が必須要件に含めているため、保存・読み込み処理を担当する ReviewCsvRepository を追加する方針にした。

Main はアプリの起動、メニュー表示、ユーザー入力、保存・読み込み処理の呼び出しを担当する。  
ReviewItem は復習項目1件分のデータを保持する。  
ReviewManager は複数のReviewItemをメモリ上で管理し、追加、一覧取得、検索、更新、集計を担当する。  
ReviewCsvRepository はCSVファイルへの保存と読み込みを担当する。

また、クラス構成とデータ保存の流れを整理するため、docs/design.md を作成した。

## 2026-05-13

### 今日やること

Phase 2-3 として、コンソールに表示するメニュー構成を決める。

### 作業内容

アプリ起動時に表示するメニューを検討した。  
今回のアプリは、復習した内容を保管し、後から見返せるようにすることを目的としているため、登録、一覧表示、詳細表示、編集、成果サマリー表示、保存して終了の6項目に整理した。

当初は「理解度を更新する」機能を想定していたが、実際に使用する際にはカテゴリ、タイトル、メモなども修正したくなる可能性があるため、「復習項目を編集する」機能としてまとめる方針にした。

また、削除機能については、入力ミスや不要な項目の整理に役立つ可能性があるため、追加要件として残すことにした。  
ただし、MVPでは必須とせず、余裕があれば実装する扱いにする。

CSVファイルの読み込みは起動時に自動で行い、保存は終了時に行う方針にした。  
そのため、CSV読み込み用のメニューは用意しないことにした。
