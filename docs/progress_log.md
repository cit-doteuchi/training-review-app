## 2026-05-11

### 今日やること

ReviewItemに持たせる項目を決める。

### 作業内容
ReviewItemに持たせる項目を検討し、復習内容を保管する置き場としてシンプルに運用できるよう、項目を決定した。
当初候補にあった「復習済みかのチェック」「できるようになったことの要約」「次に勉強すること」は、復習タスクの進捗管理や成果報告を細かく行うための項目であり、今回の「復習内容を保管する」という目的には必須ではないため採用しないことにした。

#### ReviewItemに持たせる項目

| 項目名 | 型 | 必須 | 内容 | 使用目的 |
|---|---|---|---|---|
| id | int | 必須 | 復習項目を識別する番号 | 詳細表示・更新対象の指定に使う |
| date | LocalDate | 必須 | 復習項目を登録した日 | 日付別に見返すために使う |
| category | String | 必須 | Java、SQL、HTML/CSSなどの分類 | カテゴリ別に整理するために使う。カテゴリ基準が不明なのでStringで管理 |
| title | String | 必須 | 復習項目のタイトル | 一覧表示で内容を把握するために使う |
| memo | String | 任意 | 復習内容のメモ | 後から内容を見返すために使う |
| understanding | int | 任意 | 理解度。1〜5で管理する | 必要に応じて理解度別に振り返るために使う |
